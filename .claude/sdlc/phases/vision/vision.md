# Vision: todo-list

## 1. Назначение

Артефакт продвигает альфы Opportunity и Stakeholders.
Метод — одностраничное описание проблемы и цели без формализации стейкхолдеров.

## 2. Привязка к фазе и методу

- Фаза: vision.
- Уровень SME: pet.
- Дисциплины: product-discovery, stakeholder-analysis.
- Инструмент: README-as-vision (см. `catalogs/method-tool-matrix.md`).

## 3. Содержание

### 3.1. Проблема

Нет компактного pet-проекта, где одновременно видны все фазы SDLC с артефактами.
Обучающие материалы по методу обычно разобщены и не привязаны к единому домену.

### 3.2. Решение

Собрать учебный single-user TODO-трекер.
Пройти по нему все семь фаз SDLC с живыми артефактами в `.claude/sdlc/`.
Домен TODO выбран как минимальный; предмет вторичен, цель — демонстрация процесса.

### 3.3. Ценность для бенефициара

Бенефициар — пользователь-автор (single-user).
Получает наглядный пример SDLC на собственном опыте.
Получает рабочий локальный TODO как побочный продукт.

### 3.4. Цели

1. Пройти все 7 фаз SDLC по порядку, указанному в плагине.
2. На каждой фазе зафиксировать один артефакт в `.claude/sdlc/phases/<phase>/`.
3. Продвинуть все базовые альфы минимум до целевых состояний фазы.

### 3.5. Не-цели

- Многопользовательский режим.
- Аутентификация и авторизация.
- Масштабируемость и высокая нагрузка.
- Интеграции с внешними сервисами.
- Мобильные клиенты.

### 3.6. Стейкхолдеры

| Стейкхолдер          | Интерес                         | Противоречие                |
| -------------------- | ------------------------------- | --------------------------- |
| Пользователь-автор   | Работающий TODO и освоение SDLC | Время vs глубина проработки |
| Читатель репозитория | Понятный учебный пример         | Компактность vs полнота     |

Противоречие разрешается в пользу компактности: предмет вторичен, фаза демонстрируется минимальной полнотой артефакта.

### 3.7. Альтернативы, отвергнутые на этом уровне

1. Клон известного приложения (issue-tracker) — богаче домен, но отвлекает от SDLC.
2. Чистый методологический репозиторий без кода — исчезает фаза development.
3. Сторонний готовый pet-проект — не контролируется каркас `.claude/sdlc/`.

## 4. Трассируемость

- `traces_from`: нет (фаза входная).
- `traces_to`: будет заполнено в фазе requirements.
- Состояние альф: `.claude/sdlc/alphas.md`.
- Реестр систем: `.claude/sdlc/system-context.md`.

## 5. Критерии готовности

- Артефакт проходит `validate-artifact.sh`.
- Альфа Opportunity не ниже Value Established.
- Альфа Stakeholders не ниже Represented.
- `check-cross-refs.sh` не находит осиротевших ссылок.
- Критерий успеха проекта: все 7 фаз SDLC пройдены с артефактами.
