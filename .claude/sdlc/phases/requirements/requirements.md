## 3. Содержание

### 3.1. Функциональные истории (Functional Stories)

#### US-01 — Добавить задачу

Как пользователь, я хочу добавить задачу в список, чтобы зафиксировать дело.
**Критерий:** когда я ввожу текст и подтверждаю, задача появляется в списке со статусом «активна».

#### US-02 — Переключить статус задачи

Как пользователь, я хочу отметить задачу выполненной и вернуть её в активные.
**Критерий:** когда я переключаю задачу, её статус меняется active↔completed без потери текста.

#### US-03 — Удалить задачу

Как пользователь, я хочу удалить задачу, чтобы убрать ненужное из списка.
**Критерий:** когда я удаляю задачу, она исчезает из всех фильтров навсегда.

#### US-04 — Фильтр «активные»

Как пользователь, я хочу видеть только активные задачи, чтобы сфокусироваться на незавершённом.
**Критерий:** при выборе фильтра «активные» отображаются задачи со статусом active.

#### US-05 — Фильтр «выполненные»

Как пользователь, я хочу видеть только выполненные задачи для обзора сделанного.
**Критерий:** при выборе фильтра «выполненные» отображаются задачи со статусом completed.

#### US-06 — Фильтр «все»

Как пользователь, я хочу видеть все задачи единым списком по умолчанию.
**Критерий:** по умолчанию активен фильтр «все»; отображаются задачи любого статуса.

### 3.2. Нефункциональные требования (Non-Functional)

#### NFR-01 — Persistence через localStorage

Как пользователь, я хочу чтобы задачи сохранялись между сессиями без серверa.
**Критерий:** после перезагрузки страницы список задач и текущий фильтр восстанавливаются.

#### NFR-02 — Без регистрации и авторизации

Как пользователь, я хочу пользоваться приложением без логина и регистрации.
**Критерий:** приложение работает сразу после открытия; учётные записи отсутствуют.

### 3.3. Явные не-требования

- Синхронизация между устройствами и пользователями.
- Облачный бэкенд, сервер, API.
- Редактирование текста существующей задачи (выходит за MVP).
- Сортировка, приоритеты, дедлайны, категории.
- Экспорт/импорт данных, undo/redo.
