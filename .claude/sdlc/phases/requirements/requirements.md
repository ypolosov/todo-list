# Requirements: todo-list

## 1. Назначение

Артефакт продвигает альфы Requirements и Stakeholders.
Метод — свободный список желаемого поведения без критериев приёмки.

## 2. Привязка к фазе и методу

- Фаза: requirements.
- Уровень SME: pet.
- Дисциплины: requirements-engineering, stakeholder-analysis.
- Инструмент: Плоский TODO-список (см. `catalogs/method-tool-matrix.md`).
- Критерии приёмки на этом уровне не фиксируются; будут выведены при testing.

## 3. Содержание

### 3.1. Функциональные требования

- [ ] R-1. Добавить задачу с текстовым названием.
- [ ] R-2. Переключать статус задачи между активна и выполнена.
- [ ] R-3. Удалять задачу из списка.
- [ ] R-4. Отображать все задачи списком.
- [ ] R-5. Фильтровать список: только активные.
- [ ] R-6. Фильтровать список: только выполненные.
- [ ] R-7. Фильтровать список: все задачи без фильтра.
- [ ] R-8. Сохранять состояние задач в localStorage браузера.
- [ ] R-9. Восстанавливать состояние задач после перезагрузки страницы.

### 3.2. Нефункциональные требования

- [ ] N-1. Без регистрации и учётных записей: single-user.
- [ ] N-2. Без серверного backend: только клиентская часть.
- [ ] N-3. Работает в современном десктопном браузере.
- [ ] N-4. Все данные хранятся локально у пользователя.

### 3.3. Вне объёма (из vision)

- Многопользовательский режим.
- Синхронизация между устройствами.
- Внешние интеграции и мобильные клиенты.
- Аутентификация, авторизация, приглашения.

### 3.4. Связь со стейкхолдерами

| Стейкхолдер          | Удовлетворяется требованиями           |
| -------------------- | -------------------------------------- |
| Пользователь-автор   | R-1..R-9, N-1..N-4                     |
| Читатель репозитория | Все требования как учебный пример SDLC |

## 4. Трассируемость

- `traces_from`: `.claude/sdlc/phases/vision/vision.md` (цели 3.4 и не-цели 3.5).
- `traces_to`: будет заполнено в фазе architecture.
- Состояние альф: `.claude/sdlc/alphas.md`.

## 5. Критерии готовности

- Артефакт проходит `validate-artifact.sh`.
- Альфа Requirements не ниже Bounded: объём зафиксирован, границы определены.
- Альфа Stakeholders не ниже Involved: стейкхолдеры связаны с требованиями.
- `check-cross-refs.sh` не находит осиротевших ссылок.
