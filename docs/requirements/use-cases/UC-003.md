# UC-003: Поиск и фильтрация задач

## Actors

- Пользователь (единственный, без аутентификации)

## Preconditions

- Приложение открыто в браузере
- В localStorage существует хотя бы одна задача

## Main Flow

### Поиск задач по тексту

1. Пользователь вводит текст в поле поиска
2. Система фильтрует список задач в реальном времени
3. Система отображает только задачи, содержащие введенный текст в названии
4. При очистке поля поиска система отображает все задачи

### Фильтрация по статусу выполнения

1. Пользователь выбирает фильтр: Все / Активные / Выполненные
2. Система отображает задачи, соответствующие выбранному статусу

### Фильтрация по категории

1. Пользователь выбирает категорию: Все / Работа / Дом
2. Система отображает задачи выбранной категории
   (пересекается с UC-002, просмотр по категории)

### Комбинированная фильтрация

1. Пользователь применяет несколько фильтров одновременно
   (текст + статус + категория)
2. Система отображает задачи, удовлетворяющие всем выбранным критериям

## Alternative Flows

### AF-1: Нет результатов поиска

1. Ни одна задача не соответствует критериям
2. Система отображает сообщение "Задачи не найдены"

### AF-2: Пустой список задач

1. В системе нет задач
2. Поиск и фильтрация недоступны или отображают пустой результат
   с предложением создать первую задачу

## Postconditions

- Отображенный список задач соответствует заданным критериям
- Исходные данные не изменены (поиск и фильтрация -- операции чтения)
- Состояние фильтров сохраняется до явного сброса пользователем
