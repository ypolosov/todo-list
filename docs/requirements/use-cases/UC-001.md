# UC-001: Управление задачами

## Actors

- Пользователь (единственный, без аутентификации)

## Preconditions

- Приложение открыто в браузере
- localStorage доступен

## Main Flow

### Создание задачи

1. Пользователь нажимает кнопку создания задачи
2. Система отображает форму ввода
3. Пользователь вводит название задачи
4. Пользователь подтверждает создание
5. Система сохраняет задачу в localStorage
6. Система отображает задачу в списке

### Редактирование задачи

1. Пользователь выбирает существующую задачу
2. Система отображает форму редактирования с текущими данными
3. Пользователь изменяет название задачи
4. Пользователь подтверждает изменения
5. Система обновляет задачу в localStorage
6. Система отображает обновленную задачу в списке

### Удаление задачи

1. Пользователь инициирует удаление задачи
2. Система запрашивает подтверждение
3. Пользователь подтверждает удаление
4. Система удаляет задачу из localStorage
5. Система обновляет список задач

### Отметка выполнения

1. Пользователь отмечает задачу как выполненную (или снимает отметку)
2. Система обновляет статус задачи в localStorage
3. Система визуально отражает изменение статуса

## Alternative Flows

### AF-1: Пустое название при создании/редактировании

1. Пользователь оставляет название пустым и подтверждает
2. Система отображает сообщение об ошибке валидации
3. Форма остается открытой для исправления

### AF-2: Отмена удаления

1. На шаге подтверждения удаления пользователь отменяет действие
2. Задача остается без изменений

### AF-3: localStorage недоступен или переполнен

1. Система обнаруживает ошибку при сохранении
2. Система отображает сообщение об ошибке
3. Данные формы сохраняются для повторной попытки

## Postconditions

- При создании: новая задача сохранена и отображается в списке
- При редактировании: задача обновлена в хранилище и на экране
- При удалении: задача удалена из хранилища и из списка
- При отметке: статус выполнения обновлен
