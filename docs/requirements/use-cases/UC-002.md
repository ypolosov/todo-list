# UC-002: Категоризация задач

## Actors

- Пользователь (единственный, без аутентификации)

## Preconditions

- Приложение открыто в браузере
- Существуют предустановленные категории: Работа, Дом

## Main Flow

### Назначение категории при создании задачи

1. Пользователь создает новую задачу (UC-001, создание)
2. Система отображает выбор категории с вариантами: Работа, Дом
3. Пользователь выбирает категорию
4. Система сохраняет задачу с привязкой к выбранной категории

### Изменение категории существующей задачи

1. Пользователь редактирует задачу (UC-001, редактирование)
2. Система отображает текущую категорию задачи
3. Пользователь изменяет категорию
4. Система обновляет привязку задачи в localStorage

### Просмотр задач по категории

1. Пользователь выбирает категорию (Работа или Дом)
2. Система отображает только задачи выбранной категории
3. Пользователь может переключаться между категориями

## Alternative Flows

### AF-1: Задача без категории

1. Пользователь не выбирает категорию при создании
2. Система назначает категорию по умолчанию или требует выбор
   (решение за этапом проектирования)

## Postconditions

- Каждая задача привязана к одной из предустановленных категорий
- Задачи корректно отображаются при фильтрации по категории
- Изменение категории сохранено в localStorage
