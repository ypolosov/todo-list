# Todo List -- Обзор требований

## Назначение

Личный инструмент для управления задачами. Позволяет создавать, редактировать,
удалять и отмечать выполнение задач, распределяя их по предустановленным категориям
(Работа, Дом). Поддерживает поиск и фильтрацию.

## Бизнес-контекст

| Параметр              | Значение                          |
|-----------------------|-----------------------------------|
| Тип продукта          | Личный инструмент                 |
| Количество пользователей | Один                           |
| Аутентификация        | Отсутствует                       |
| Уведомления           | Не требуются                      |
| Планы развития        | Нет (только MVP)                  |

## Функциональный охват

- **UC-001** -- Управление задачами (CRUD + отметка выполнения)
- **UC-002** -- Категоризация задач (предустановленные списки: Работа, Дом)
- **UC-003** -- Поиск и фильтрация задач

## Технический стек

| Компонент   | Решение                     |
|-------------|-----------------------------|
| Платформа   | Web-приложение              |
| Frontend    | React                       |
| Backend     | Требуется уточнение (CRN-001) |
| Хранение    | localStorage                |
| Режим работы| Только оффлайн              |

## Ограничения

- **CON-001** -- Только оффлайн, данные в localStorage
- **CON-002** -- Без аутентификации, однопользовательский режим

## Открытые вопросы

- **CRN-001** -- Архитектурное противоречие: React + бэкенд при полностью
  оффлайн-режиме с localStorage. Необходимо уточнить, нужен ли бэкенд
  или достаточно SPA.
