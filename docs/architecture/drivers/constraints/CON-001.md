# CON-001: Только оффлайн, данные в localStorage

## Description

Приложение работает исключительно в оффлайн-режиме. Все данные хранятся
в localStorage браузера. Никакие сетевые запросы не выполняются.
Облачная синхронизация не предусмотрена.

## Rationale

Пользователь явно указал, что приложение должно работать без интернета,
без облака. localStorage выбран как простейший механизм персистентного
хранения на клиентской стороне.

## Implications

- Объем данных ограничен лимитом localStorage (~5-10 MB в зависимости от браузера)
- Данные привязаны к конкретному браузеру и домену
- Очистка данных браузера приведет к потере всех задач
- Резервное копирование данных возможно только вручную (экспорт/импорт -- вне MVP)
- Бэкенд-сервер не может использовать серверную БД для хранения
- При выборе архитектуры с бэкендом -- бэкенд становится лишь статическим
  сервером для раздачи SPA (см. CRN-001)
