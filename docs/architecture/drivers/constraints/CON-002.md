# CON-002: Без аутентификации, однопользовательский режим

## Description

Приложение рассчитано на одного пользователя. Аутентификация и авторизация
отсутствуют. Все данные в localStorage принадлежат единственному пользователю.

## Rationale

Это личный инструмент. Нет необходимости в разграничении доступа,
сессиях, токенах или ролевой модели.

## Implications

- Не требуется модель пользователя (User entity)
- Не требуется middleware аутентификации
- Не требуется шифрование данных на стороне клиента (данные доступны
  любому, кто имеет доступ к браузеру)
- Упрощается модель данных: задачи не привязаны к user ID
- Сокращается объем разработки и тестирования
